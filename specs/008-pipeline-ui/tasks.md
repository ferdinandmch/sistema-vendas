## Phase 3: User Story 1 — Visualizar board com colunas e cards (Priority: P1)

**Goal**: Usuário autenticado acessa `/pipeline` e vê stages como colunas (ordenadas por `position`) com deals distribuídos como cards no `stage_id` correto.

**Independent Test**: Criar 3 stages e 5 deals distribuídos via API. Acessar `/pipeline`. Verificar 3 colunas na ordem de `position` com deals corretos em cada coluna.

### Tests for User Story 1

- [X] T011 [P] [US1] Create unit test `tests/unit/pipeline/group-deals-by-stage.test.ts` — import `groupDealsByStage` from `@/lib/pipeline/group-deals` — test: (a) `groupDealsByStage(deals)` groups correctly by `stageId`, (b) deal with unknown `stageId` is absent from result when board iterates known stages (`Record` simply has no key for it — BR-005), (c) `groupDealsByStage([])` returns `{}`

### Implementation for User Story 1

- [X] T012 [P] [US1] Create `components/pipeline/DealCard.tsx` — receive `deal: Deal` as prop — render shadcn `<Card>` with `<CardContent>`: `name` (title), `companyName` (subtitle), `value` (formatted currency), `status` as shadcn `<Badge>` (variant: default/secondary/destructive per active/won/lost) — add `data-deal-id={deal.id}` on root element (FR-009) — no `onClick` (FR-011)
- [X] T013 [P] [US1] Create `components/pipeline/PipelineColumn.tsx` — receive `stage: Stage` and `deals: Deal[]` as props — render column with stage `name` as header and a list of `<DealCard>` for each deal
- [X] T013b [P] [US1] Create `lib/pipeline/group-deals.ts` — export `groupDealsByStage(deals: Deal[]): Record<string, Deal[]>` pure function — group by `deal.stageId` — deals with unmatched stageId are naturally omitted when board iterates `stages.map(s => result[s.id] ?? [])` (BR-005)
- [X] T014 [US1] Create `components/pipeline/PipelineBoard.tsx` as `"use client"` — use `useQuery(stageKeys.list(), fetchStages)` and `useQuery(dealKeys.list(), fetchDeals)` independently — import `groupDealsByStage` from `@/lib/pipeline/group-deals` — apply with `useMemo` — render horizontally scrollable container with `stages.sort((a, b) => a.position - b.position).map(stage => <PipelineColumn>)` — pass `dealsByStage[stage.id] ?? []` to each column
- [X] T015 [US1] Update `app/(private)/pipeline/page.tsx` — keep `requireAuthenticatedUser()` server-side auth check — replace placeholder JSX with `<PipelineBoard />` — remove inline styles — this file remains a Server Component (no `"use client"`)

**Checkpoint**: Board com colunas e cards visível em `/pipeline`. Ordenação por `position` correta. Deals na coluna certa.
